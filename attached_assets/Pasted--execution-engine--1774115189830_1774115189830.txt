المطلوب: إصلاح سلوك execution_engine بشكل نهائي ومنع اختيار الأدوات الخاطئة

========================================

1. تعريف أنواع الطلبات (إجباري)
   ========================================

يجب على الوكيل تصنيف الطلب قبل التنفيذ:

TYPE 1: UI_CHANGE

* تعديل واجهة
* نصوص
* ألوان
* CSS

TYPE 2: BACKEND_CHANGE

* API
* logic
* server

TYPE 3: READ_ONLY

* استعلام
* معلومات
* تحليل

TYPE 4: DATABASE

* استعلام أو تعديل قاعدة بيانات

========================================
2) مسارات التنفيذ حسب النوع
===========================

A) UI_CHANGE:

* get_page_structure
* search_text
* read_file
* edit_component

---

B) BACKEND_CHANGE:

* search_text
* read_file
* edit_component

---

C) READ_ONLY:

* read_file
* list_files
* search_text

ممنوع:

* أي تعديل

---

D) DATABASE:

* db_query (قراءة فقط)

إذا لزم:

* run_sql (بموافقة)

========================================
3) قواعد الأدوات (إجباري)
=========================

* search_text و read_file هي الأدوات الأساسية
* list_files للاستكشاف فقط

ممنوع استخدام:

* run_command
* exec_command

إلا إذا:

* جميع الأدوات الأخرى فشلت
* وتم تبرير السبب

========================================
4) DOM إلزامي فقط للواجهة
=========================

* DOM مطلوب فقط لـ UI_CHANGE
* ممنوع طلب DOM في backend

========================================
5) منع حلقات الموافقات
======================

إذا احتاجت العملية أكثر من:
1 approval

يجب:

* تغيير الاستراتيجية فورًا
* استخدام أدوات لا تحتاج موافقة

========================================
6) منع سلوك CLI
===============

أضف:

"أنت لا تستخدم terminal.
ممنوع:

* grep
* find
* bash
* shell thinking

استخدم أدوات النظام فقط"

========================================
7) حد العمليات
==============

* max search = 3
* max tools بدون edit = 4

إذا تجاوز:

* توقف فورًا
* اطلب توجيه

========================================
8) التنفيذ الإجباري
===================

ممنوع:

* "سأفعل"
* "دعني أبحث"

يجب:

* تنفيذ مباشر

========================================
9) التحقق الإجباري
==================

بعد edit:

* screenshot_page
* verify_production
