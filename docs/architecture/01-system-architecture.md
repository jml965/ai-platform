# المعمارية العامة للنظام — System Architecture

## نظرة عامة

منصة بناء المواقع بالذكاء الاصطناعي — يصف المستخدم الموقع الذي يريده بلغته الطبيعية (عربي أو إنجليزي)، ويقوم النظام بتوليد الكود وتشغيله ومعاينته مباشرة.

---

## مخطط الطبقات (Layered Architecture)

```
┌─────────────────────────────────────────────────────────────┐
│                    طبقة العرض (Presentation)                │
│  React + Vite + i18n (AR/EN) + RTL/LTR                     │
│  - لوحة التحكم (Dashboard)                                  │
│  - محرر المحادثة (Chat Editor)                               │
│  - نافذة المعاينة (Live Preview - iframe)                    │
│  - عارض سجل التنفيذ (Execution Log Viewer)                   │
│  - صفحات الفوترة والفريق                                    │
└──────────────────────┬──────────────────────────────────────┘
                       │ HTTPS / REST API
┌──────────────────────▼──────────────────────────────────────┐
│                    طبقة الـ API (API Gateway)                │
│  Express 5 + Node.js                                        │
│  - المصادقة (Auth Middleware)                                │
│  - الصلاحيات (RBAC Middleware)                               │
│  - التحقق من الطلبات (Zod Validation)                       │
│  - حدود الإنفاق (Rate & Cost Limiter)                       │
│  Routes:                                                    │
│  ├── /api/auth/*        المصادقة                             │
│  ├── /api/projects/*    المشاريع                             │
│  ├── /api/build/*       أوامر البناء                         │
│  ├── /api/agents/*      حالة الوكلاء                         │
│  ├── /api/tokens/*      استهلاك التوكن                       │
│  ├── /api/billing/*     الفوترة والاشتراكات                  │
│  ├── /api/teams/*       إدارة الفرق                          │
│  └── /api/healthz       فحص الصحة                            │
└──────────────────────┬──────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────────┐
│                 طبقة محرك الوكلاء (Agent Engine)             │
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ Code Gen    │  │ Code Review │  │ Fix Agent   │         │
│  │ Agent       │  │ Agent       │  │             │         │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘         │
│         │                │                │                 │
│  ┌──────▼────────────────▼────────────────▼──────┐          │
│  │         محرك التنفيذ (Execution Engine)        │          │
│  │  - توزيع المهام على الوكلاء                    │          │
│  │  - إدارة قائمة الانتظار (Redis + BullMQ)       │          │
│  │  - تطبيق دستور الوكلاء (Agent Constitution)    │          │
│  │  - تسجيل كل عملية في سجل التنفيذ              │          │
│  └───────────────────────────────────────────────┘          │
└──────────────────────┬──────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────────┐
│                 طبقة الذكاء الاصطناعي (AI Layer)             │
│  - OpenAI / Anthropic API (قابل للتبديل)                    │
│  - نموذج قوي للمهام المعقدة (GPT-4 / Claude)               │
│  - نموذج خفيف للمهام البسيطة (GPT-3.5 / Haiku)            │
│  - Prompt Templates لكل نوع وكيل                           │
└──────────────────────┬──────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────────┐
│                    طبقة البيانات (Data Layer)                │
│  PostgreSQL + Drizzle ORM                                   │
│  جداول:                                                     │
│  ├── users             المستخدمون                           │
│  ├── teams             الفرق                                │
│  ├── team_members      أعضاء الفرق                          │
│  ├── projects          المشاريع                             │
│  ├── project_files     ملفات المشروع                        │
│  ├── build_tasks       مهام البناء                           │
│  ├── execution_logs    سجل التنفيذ                          │
│  ├── token_usage       استهلاك التوكن                       │
│  ├── plans             خطط الاشتراك                         │
│  ├── subscriptions     اشتراكات المستخدمين                  │
│  ├── invoices          الفواتير                              │
│  └── credits_ledger    سجل الرصيد                           │
└──────────────────────┬──────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────────┐
│                 طبقة التخزين (Storage Layer)                 │
│  - ملفات المشاريع المولَّدة: Object Storage                  │
│  - المعاينة المباشرة: Sandbox Environment (iframe)          │
└─────────────────────────────────────────────────────────────┘
```

## تدفق البيانات الرئيسي (Main Data Flow)

```
المستخدم يكتب طلب بناء موقع
        │
        ▼
┌──────────────────┐
│ Frontend (React) │ ──► POST /api/build/start
└──────────────────┘
        │
        ▼
┌──────────────────┐
│ API Server       │ ──► يتحقق من الصلاحيات والحدود
└──────────────────┘
        │
        ▼
┌──────────────────┐
│ Execution Engine │ ──► يقسّم الطلب إلى مهام صغيرة
└──────────────────┘
        │
        ├──► مهمة 1: توليد HTML ──► Code Gen Agent ──► AI API
        ├──► مهمة 2: توليد CSS  ──► Code Gen Agent ──► AI API
        ├──► مهمة 3: توليد JS   ──► Code Gen Agent ──► AI API
        │
        ▼
┌──────────────────┐
│ Code Review Agent│ ──► يراجع الكود المولَّد
└──────────────────┘
        │
        ├── نجح ──► حفظ الملفات في DB + تحديث المعاينة
        │
        └── فشل ──► Fix Agent ──► إصلاح الجزء المتأثر فقط
                        │
                        └──► إعادة المراجعة (حد 3 محاولات)
        │
        ▼
┌──────────────────┐
│ Live Preview     │ ──► المستخدم يرى الموقع فوراً
└──────────────────┘
```

## مبادئ المعمارية

| المبدأ | الشرح |
|--------|-------|
| **الفصل بين الطبقات** | كل طبقة مستقلة ولها واجهة محددة |
| **الوكيل المحدود** | كل وكيل يعمل ضمن مهمة واحدة فقط |
| **التوليد التدريجي** | الملفات تُنشأ واحداً تلو الآخر وليس دفعة واحدة |
| **الإصلاح الجزئي** | عند وجود خطأ، يُصلح الجزء المتأثر فقط |
| **الشفافية** | كل عملية تُسجَّل في سجل التنفيذ |
| **التحكم بالتكلفة** | حد توكن لكل وكيل ولكل مشروع |
| **قائمة مهام موثوقة** | Redis + BullMQ لضمان عدم ضياع المهام |
| **ثنائي اللغة** | جميع الواجهات تدعم العربية (RTL) والإنجليزية (LTR) |
